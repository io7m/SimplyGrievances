5.35. simplified-accounts-database

5.35.1. Issues

5.35.1.1. At-Least-One Account Limit

1
For historical reasons, the accounts database requires that at least one account must exist at any given time. This is because the support for multiple accounts was added (badly) in version 2.0 of the application, and was rewritten from the ground up for the Library For All fork of the codebase without there being resources available to redesign the UI to properly support multiple (or zero) accounts. The simplest way to avoid having to do the UI work was to ensure that at least one account was always available for the UI to display.
2
Issue class: DESIGN. Severity: Major.
3
Relevant tickets: SMA-173.

5.35.2. Module Link

1
The module can be found at simplified-accounts-database.
