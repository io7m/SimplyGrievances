5.26. org.librarysimplified.audiobook.views

5.26.1. Issues

5.26.1.1. PlayerFragment Uses UIThread

1
The PlayerFragment class uses the fundamentally flawed UI thread API.
2
Issue class: UITHREAD. Severity: Critical.

5.26.1.2. Activity Listeners

1
The hosting Activity must communicate with Fragments using old-style callback functions. This should be replaced with communcation via a ViewModel (the Audio Book library predates ViewModels).
2
Issue class: STYLE. Severity: Minor.

5.26.1.3. Designed By Committee TOC

1
The design for the table of contents UI was essentially put together via a committee and has numerous shortcomings. Firstly, it is aesthetically ugly. Secondly, there is no way for users to see details of any errors that may occur when downloading chapters. This harms the ability of developers to debug issues.
2
Issue class: DESIGN. Severity: Major.

5.26.1.4. Player Colors Taken From Theme

1
The Audio Book player UI takes colors from the current application theme. This may present difficulties if the application wants to move to a partially dynamic, neutral color scheme.
2
Issue class: DESIGN. Severity: Minor.

5.26.1.5. Audio Player Can Delete Accounts

1
A serious and as-yet-unexplained bug appears to be present in the audio book player where exiting an audio book can delete or corrupt the application account containing the audio book.
2
Issue class: BUG. Severity: Critical.
3
Relevant issues: https://www.notion.so/lyrasis/5694f5b2d40c4baea0d97782a3141cef.

5.26.2. Module Link

1
The module can be found at org.librarysimplified.audiobook.views.
