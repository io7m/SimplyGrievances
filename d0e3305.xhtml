5.53. simplified-books-borrowing

5.53.1. Issues

5.53.1.1. Book Subtask Ordering

1
Subtasks in the book borrowing code do not have any kind of priority ordering. A task is selected for execution if it matches the required media type, and the first matching subtask is used. This means that if there are multiple subtasks that support a given media type, the subtask is selected nondeterministically. Ideally, subtasks would specify some kind of priority ordering to allow overriding another known task.
2
Issue class: DESIGN. Severity: Minor.
3
Relevant issues: SMA-153.

5.53.2. Module Link

1
The module can be found at simplified-books-borrowing.
