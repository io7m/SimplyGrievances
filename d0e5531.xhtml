5.127. simplified-viewer-audiobook

5.127.1. Issues

5.127.1.1. Uses UIThread API

1
The module uses the fundamentally flawed UI thread API.
2
Issue class: UITHREAD. Severity: Critical.

5.127.1.2. Player Is Activity-Scoped

1
The audio book player instances are stored in the hosting activity. This often means that audio will stop playing if the activity is destroyed by the system when the activity is in the background.
2
Issue class: LIFECYCLE. Severity: Major.

5.127.1.3. Problematic Bookmark Storage

1
The audio player appears to have problems storing bookmarks. The most likely cause of this issue is a combination of the application eagerly storing bookmarks in response to player position update events, and the underlying player publishing invalid position updates. A first approach should be to implement a stream of carefully-vetted events that represent the one and only means by which the application should be provoked into storing a bookmark.
2
Issue class: BUG. Severity: Major.
3
Relevant issues: SMA-101, SMA-114.

5.127.1.4. Activity Listeners

1
Due to the design of the Audio Book API, the hosting Activity must communicate with Fragments using old-style callback functions. This should be replaced with communcation via a ViewModel (the Audio Book library predates ViewModels).
2
Issue class: STYLE. Severity: Minor.

5.127.2. Module Link

1
The module can be found at simplified-viewer-audiobook.
